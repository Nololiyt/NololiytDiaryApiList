# 总则
这里是Nololiyt Diary的公开API列表，您可以就此为这个项目进行客户端开发。
## 客户端开发注意事项
+ 务必仔细阅读此说明。
+ 在用户登录时，客户端应该将最新的协议呈现（除非所登录的用户已阅读过）并要求其同意。若用户使用某一无该功能的客户端而导致损失，应当由客户端开发者负责。
+ 为了传输安全，尽可能使用https而非http。若用户使用某一提供http访问的客户端而导致损失，应当由客户端开发者负责。
+ 客户端开发者可以向Nololiyt官方申请（联系方式见ContactWay.md），经过审核，可被作为“推荐的第三方版本”之类在官网进行宣传。
## Api相关
+ Api列表位于ControllersAndActions文件夹下。服务端使用MVC，有统一的访问链接模板如下：api/[controller]/[action]。其中文件名和二级标题标识[controller]，三级标题标识[action]。
+ 除非有特殊说明的，有参使用Post方法，无参使用Get方法。
+ 参数、返回值类型中出现的自定义类型具体定义见Entity文件夹。其中存有对应C#代码（没有逻辑代码，较易于理解）。无参或无返回值使用void表示。
+ 枚举类型返回内容直接为数字。
+ 除非有特殊说明的，参数一律FromBody，建议使用json形式（其他形式不保证可行）。
+ 登录使用Session方式。调用接口一般都需要登录，不用登录的将另作说明。
+ 密码可以直接发送，服务端将计算hash后储存。如果认为传输过程不安全，也可以计算hash并使用base64等方式编码后传输。但注意不允许超过30字符（中文字符算1字符）。
